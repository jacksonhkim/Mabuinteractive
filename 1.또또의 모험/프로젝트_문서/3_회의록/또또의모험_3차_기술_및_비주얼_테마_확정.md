# [마부 인터랙티브] '또또의 모험' 3차 기술 및 비주얼 테마 확정 회의록

**날짜:** 2026-02-10
**시간:** 23:08 (오후 11시 08분)
**회의 주제:** 웹 HTML 기반 기술 스택 및 픽셀 아트(Pixel Art) 디자인 스타일 확정
**참석자:** 개발 총괄(또또), 시스템 기획자(뽀), 아트 디렉터(한순이), 웹 개발자, 캐릭터 디자이너, 테크 PM 외

## 1. 개요
'또또의 모험'의 성공적인 런칭을 위해 클라이언트 구동 플랫폼을 **Web HTML(Canvas)**로, 비주얼 핵심 컨셉을 **픽셀 아트(Pixel Art/Dot)**로 최종 확정하고 이에 따른 각 팀의 대응 전략을 논의함.

---

## 2. 부서별 토론 및 결정 사항

### [웹 개발팀 - 웹 HTML/Canvas 채택]
*   **발표:** "설치 없이 모든 브라우저에서 즉시 실행 가능한 **HTML5 Canvas** 기술을 사용합니다. Vanilla JS/TS와 가벼운 렌더링 라이브러리를 결합해 최저 사양 스마트폰에서도 60fps를 방어하겠습니다."
*   **또또(총괄):** "좋습니다. 액션 게임인 만큼 지연 시간(Input Lag) 최소화에 모든 기술적 역량을 집중해 주세요."

### [디자인팀 - 픽셀 아트(Pixel Art) 비주얼]
*   **한순이(AD):** "텐가이의 고전미를 현대적으로 재해석하기 위해 **고해상도 픽셀 아트** 스타일을 채택합니다. 단순히 옛날 도트 느낌이 아니라, 현대적인 컬러 팔레트와 부드러운 애니메이션 프레임을 추가해 '뉴트로' 감성을 극대화하겠습니다."
*   **캐릭터 디자이너:** "픽셀 단위의 정교한 애니메이션을 통해 캐릭터의 타격감을 손끝으로 전달하겠습니다. 3D 전환 시에도 이 픽셀 감성을 살릴 수 있는 텍스처 베이킹 기법도 함께 연구 중입니다."

### [기획팀 - 데이터 및 시스템 최적화]
*   **테크 기획자:** "웹 HTML 기반이므로 픽셀 에셋의 용량 관리가 핵심입니다. 아틀라스(Atlas) 최적화를 통해 드로우콜을 최소화하고, 초기 로딩 속도를 3초 이내로 단축하는 데이터 구조를 설계하겠습니다."
*   **전투 기획자:** "픽셀 아트는 '픽셀 퍼펙트' 판정이 가능합니다. 텐가이보다 더 정교한 히트박스 설계를 통해 액션의 신뢰도를 높이겠습니다."

### [QA 및 기술 관리]
*   **테크 PM:** "웹 브라우저의 파편화(Chrome, Safari 등)가 리스크입니다. 픽셀 디자인이 해상도별로 뭉개지지 않도록 정수배 스케일링(Integer Scaling) 기칙을 수립해 주세요."
*   **QA 담당자:** "저사양 모바일 웹 브라우저의 메모리 누수를 최우선으로 검증하겠습니다. 특히 픽셀 스프라이트 애니메이션이 많아질 때의 부하를 집중 체크하겠습니다."

## 3. 최종 확정 (마무리 보고)

1.  **플랫폼:** Web HTML (HTML5 Canvas 기반 웹 앱)
2.  **그래픽 스타일:** 현대적 감각의 고퀄리티 픽셀 아트 (Full Pixel Animation)
3.  **프레임 목표:** 모든 타겟 기기에서 안정적인 60fps 유지
4.  **다음 단계:** 확정된 픽셀 스타일 가이드에 맞춘 토토(주인공) 캐릭터 작업 착수 및 웹 코어 엔진 통합 테스트
