## 1. 픽셀 아트 제작 기술 규격 (Technical Parameters)

| 구분 | 상세 규격 (Gemini API Configuration Ready) |
| :--- | :--- |
| **1. 해상도 및 비율** | **네이티브 320x180 px** (16:9 비율 유지) |
| **2. 확대 배율 지침** | 업스케일링 시 보간 없이 **4x Point Sampling (Integer Scaling)** 필수 |
| **3. 픽셀 퍼펙트 법칙** | 45도 대각선 배치 시 'Doubles' 방지, 곡선 표현 시 픽셀 계단 현상(Jaggies) 최소화 |
| **4. 캐릭터 크기 큐** | **UI/선택창: 128x128 px** (Ultra HD 리뉴얼) / **인게임: 64x64 px** (아케이드 규격 상향) |
| **5. 색상 팔레트** | **Vibrant Neo-Retro Palette** 활용. 고채도와 대비 중심의 색 설정 |
| **6. 리소 제약** | 캐릭터 오브젝트당 독자적인 **16색 인덱스 팔레트** 할당 (투명색 1번 고정) |
| **7. 조명 시스템** | 전역 광원 [Direction: 315도(좌상단), Hard Shadows, No Dithering] |
| **8. 명암 계조** | 모든 색상은 3-tone 계조 레이어링 (Light, Mid, Dark) |
| **9. 아웃라인** | **1px Selective Outlining** (안쪽 빛을 받는 곳은 밝게, 아래쪽은 인접색의 최암부) |
| **10. 배경 레이어** | **Stage 01 픽스 배경**: `background/bg_PixelSky_layer01~03.png` 전용 에셋 활용 |
| **11. 가시성 가이드** | 탄환 및 아이템 외곽에 **Pure White Glowing Border** 추가 (UI 가독성 확보) |
| **12. 애니메이션 프레임** | Idle: 4~6fps, Fast Action: 8fps 이상, 프레임 간 픽셀 보존 법칙 준수 |
| **13. 시스템 최적화** | Texture Atlas 방식 사용, 스프라이트간 간격 2px 투명 여백 포함 |
| **14. 렌더링 세팅** | Browser Context: `image-rendering: pixelated;` 강제 적용 |
