## 🟡 Major (주요 문제)

### 1. 윈도우 포커스 이탈 시 키 입력 고착 (Input Stuck)
*   **현상**: 플레이어가 이동 키(예: 오른쪽 화살표)를 누른 상태에서 `Alt+Tab`이나 마우스 클릭으로 게임 창 밖을 선택하면, 키 뗌(KeyUp) 신호를 받지 못해 **캐릭터가 계속 한쪽으로 이동**하는 현상 발생.
*   **재현 방법**:
    1. 게임 실행 후 오른쪽 키를 누른 채 유지.
    2. 마우스로 브라우저 외부 영역 클릭.
    3. 키보드에서 손을 떼도 캐릭터는 계속 오른쪽으로 이동함.
*   **개발팀 요청**: `window.blur` 이벤트 발생 시 모든 키 입력을 초기화(`state.keys = {}`)하는 안전 장치 추가 필요.

### 2. 고해상도 모니터 / 모바일 환경에서의 터치 및 클릭 좌표
*   **현상**: `resizeCanvas` 함수가 단순 CSS 스케일링을 사용하고 있어, 캔버스 좌표계와 클릭/터치 좌표계 간의 오차가 발생할 가능성 있음. (현재 DOM 기반 D-Pad 사용으로 큰 문제는 없으나 확인 필요)
*   **개발팀 요청**: 터치/클릭 이벤트 핸들러에서 `getBoundingClientRect`를 사용하여 정확한 상대 좌표 계산 권장.

## 🟢 Minor (경미한 문제 / 개선 사항)

### 3. 'R' 키 재시작 시 하드 리로드 (Hard Reload)
*   **현상**: 게임 오버 후 'R' 키를 누르면 `location.reload()`가 호출되어 페이지 전체가 새로고침됨.
*   **영향**: 웹 리소스 캐싱이 되어있어 빠르긴 하나, 깜빡임이 발생하고 추후 '최고 점수 저장' 등의 기능 추가 시 데이터가 휘발될 수 있음.
*   **개발팀 요청**: `resetGame()` 함수를 만들어 상태값만 초기화하는 '소프트 리셋' 방식으로 변경 고려 (v2.0 반영 권장).

### 4. 보스 밸런스 (Emperor V)
*   **현상**: 최종 보스 'Emperor V'의 체력이 4000으로 설정됨. 기본 무기(데미지 1)로는 4000발을 맞춰야 하며, 풀 파워업 상태에서도 처치 시간이 너무 길어 지루함을 유발할 수 있음.
*   **개발팀 요청**: 보스 체력을 3000 수준으로 하향 조정하거나, 플레이어의 공격력을 상향 조정하여 쾌적한 플레이 유도.
